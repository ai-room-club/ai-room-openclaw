# Промпт для Codex CLI

Скопируй этот промпт целиком, запусти Codex CLI и вставь.

---

## Шаг 1. Подготовка (в терминале)

Codex CLI — официальный инструмент OpenAI. Если у тебя уже есть подписка **ChatGPT Plus или Pro** — ты можешь пользоваться Codex бесплатно (он включён в подписку).

1. **Установи Codex CLI:**
   ```
   npm install -g @openai/codex
   ```
   (Нужен Node.js 22+. Если нет — `brew install node` на Mac или [nodejs.org/download](https://nodejs.org/en/download) на Windows.)

2. **Залогинься через ChatGPT-подписку:**
   ```
   codex login
   ```
   Откроется браузер → войди в свой ChatGPT-аккаунт → готово.

3. **Клонируй репо с установщиком:**
   ```
   git clone https://github.com/ai-room-club/ai-room-openclaw.git
   cd openclaw-ru-installer
   ```

4. **Запусти Codex в этой папке:**
   ```
   codex
   ```

## Шаг 2. Активационный промпт (копипаста)

Скопируй следующее **целиком** в чат Codex:

```
Привет. Я хочу установить OpenClaw-агента на свой VPS через этот установщик.

Ты сейчас:

1. Прочитай @AGENTS.md — там твоя роль и правила.
2. Прочитай @RUNBOOK.md — там источник истины по установке.
3. Начни Phase 0 (preflight): задай мне по одному вопросу данные, которые тебе нужны для установки.

Я абсолютный новичок — не знаю терминала, SSH и прочих технических деталей. Объясняй мне всё простым русским языком. Задавай вопросы по одному, жди ответа, прежде чем переходить к следующему.

Начинай.
```

## Что произойдёт дальше

Codex прочитает инструкцию и начнёт диалог с тобой. Он будет:

1. **Задавать тебе вопросы** по одному.
2. **Объяснять каждый шаг** простым языком.
3. **Запускать команды на твоём сервере** — только после твоего согласия, только из готовых скриптов в `scripts/`.
4. **Отчитываться о результате** каждой фазы.

## Что тебе понадобится в процессе

- **Купленный VPS с Ubuntu 22.04+** или Debian 12+ (любой провайдер). IP и пароль root на руках.
- **Рабочий VPN** — понадобится один раз на Phase 5 (OAuth).
- **Подписка ChatGPT Plus или Pro** — у тебя уже есть (через неё же работает Codex).
- **Активный Telegram-аккаунт**.
- **~60 минут свободного времени**.

## ⚠️ Что важно понимать про безопасность

Codex получит **SSH-доступ** к твоему серверу. Он:

✅ **Может:** подключаться к серверу, запускать команды из scripts/ этого репо, читать и писать файлы на сервере.

❌ **Не может:** трогать другие сервера, устанавливать OpenClaw на твой **локальный компьютер**, импровизировать команды вне scripts/, работать без твоего подтверждения на destructive-действиях.

Все его действия описаны в `AGENTS.md` и `RUNBOOK.md`. Если увидишь, что Codex пытается делать что-то **не по RUNBOOK** — остановись и напиши «Остановись и объясни, зачем тебе это».

## Особенности работы с Codex

- **Режим подтверждения по умолчанию:** Codex спрашивает перед каждой значимой командой. Это нормально — читай что он хочет сделать и подтверждай только если понятно.
- **Без `--yolo`:** мы специально не используем автономный режим для установки. Ты видишь и контролируешь каждый шаг.
- **Если Codex «завис»** на долгой команде (например, `apt install`): подожди 1-2 минуты, не прерывай. Если реально повисло — `Ctrl+C` и напиши Codex «покажи где остановились, продолжим».

## Если что-то пойдёт не так

- **Codex завис** или отвечает странно → напиши «Покажи текущую фазу и статус». Он восстановит контекст.
- **Ошибка на сервере** → Codex сам обратится к «Failure recovery» соответствующей фазы RUNBOOK.
- **Не хочешь продолжать** → напиши «Остановись. Не делай ничего больше». Codex прервёт процесс.
- **Нужен fallback** (без AI) → см. [Notion-гайд AI Room Club](https://airoomclub.ru).
