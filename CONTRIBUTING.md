# Contributing

Этот репозиторий — установщик OpenClaw с фокусом на русскоязычную аудиторию. Мы рады помощи сообщества.

## Что можно улучшать

1. **Скрипты в `scripts/`** — найдена ошибка или способ сделать шаг надёжнее? Открывай PR.
2. **Шаблоны в `templates/soul/`** — собственные SOUL.md для новых профессий или use-case.
3. **Документация в `docs/`** — уточнения, исправления, переводы.
4. **RUNBOOK.md** — если нашёл плохо описанный этап или скрытый подвох на реальном прогоне.

## Что мы НЕ принимаем

- Изменения, которые усложняют процесс для абсолютного новичка ради «гибкости» или «оптимизации».
- Рекламные ссылки, реферальные коды в README или в самих скриптах.
- Скрипты, требующие `sudo` без явной необходимости.
- Изменения, снижающие уровень безопасности (открытые порты, root-логин, отсутствие аудита).

## Процесс

1. Открой issue с описанием проблемы или предложения **до** большой работы.
2. Для кода: fork → branch → PR. Опиши изменения и результат тестирования.
3. Для документации и шаблонов SOUL: прямой PR допустим, но желательно сначала issue.
4. Обязательно **тестируй на чистом VPS** (без существующих инсталляций OpenClaw) перед отправкой PR со скриптами.

## Стандарты кода

- **Shell-скрипты:** `#!/usr/bin/env bash`, `set -euo pipefail`, без интерактивности если не обоснована, все переменные — через env или явные параметры.
- **Markdown:** русский контент, английские термины в скобках при первом упоминании.
- **Комментарии в скриптах:** на русском, объясняют «почему», а не «что».

## Тестирование

Каждый скрипт должен быть протестирован минимум на:

- Ubuntu 22.04 LTS.
- Ubuntu 24.04 LTS.
- Свежий VPS от любого провайдера (без модификаций, Ubuntu 22/24 из коробки).

Результат прогона (скриншот или лог) — приложи к PR.

## Обратная связь без PR

Если у тебя нет возможности сделать PR, но ты нашёл проблему:

- Открой [issue](https://github.com/ai-room-club/ai-room-openclaw/issues) с описанием.
- Или напиши в Discord-сервер AI Room Club (ссылка в README).

## Code of Conduct

Общаемся уважительно. Никаких оскорблений, политических дискуссий, флейма. Фокус на технической работе.

Нарушения → бан без предупреждения. Модерация — на стороне авторов репозитория.

## Лицензия вкладов

Отправляя PR, ты соглашаешься, что твой вклад лицензируется под MIT (как и весь репозиторий).
